# Free Deployment

This app can run on Render's free web service tier.

## Render

1. Push this project to a GitHub repository.
2. Go to Render and create a new Blueprint.
3. Connect the repository.
4. Render will detect `render.yaml`.
5. Deploy the `history-hunt-ph` service.

The app uses:

- Build command: `npm install`
- Start command: `npm start`
- Environment variables:
  - `DATA_STORE=mysql`
  - `DB_HOST`
  - `DB_PORT`
  - `DB_USER`
  - `DB_PASSWORD`
  - `DB_NAME`
  - `DB_SSL`
  - `JWT_SECRET` generated by Render

## MySQL Setup

Create a MySQL database first, then run `schema.sql` on that database.

Use these Render environment variables:

- `DATA_STORE=mysql`
- `DB_HOST`: your MySQL host
- `DB_PORT`: usually `3306`
- `DB_USER`: your MySQL username
- `DB_PASSWORD`: your MySQL password
- `DB_NAME`: your MySQL database name
- `DB_SSL`: use `true` for hosted MySQL providers that require SSL, otherwise `false`

## Important Free Hosting Note

Do not use `DATA_STORE=file` on Render if you need permanent accounts and leaderboard data.
File storage can reset after redeploys, restarts, or idle sleep. Use MySQL for persistent records.
